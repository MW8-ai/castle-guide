# ADR-0001: Storage strategy (local-first)

- **Status:** Accepted  
- **Date:** 2026-07-09  
- **Deciders:** Phase 0 architecture (owner-approved)

## Context

Castle Guide stores highly sensitive home data (inventories, serials, photos, shutoff locations). The constitution requires local-first operation, no forced accounts, full export/delete, and no telemetry. Multi-device use must work without a cloud dependency on day one.

## Decision

1. **Structured user data** lives in **IndexedDB** behind a repository API in `src/storage/` (implementation: Dexie or `idb` in Phase 1).
2. **Binary blobs** (photos, PDFs) use **OPFS** when available, with IndexedDB blob store fallback.
3. **Portability** via **ZIP export/import** containing `manifest.json` (`schemaVersion`), profile/property JSON, and `media/`.
4. **Shipped content** is static JSON under `/data`, validated by JSON Schema in CI; not mixed into user IDB except as read-only caches if needed.
5. **AI keys** stored only in browser settings; **excluded from default export**.
6. **Cloud sync / accounts** remain BLUEPRINT (`HUMAN_DIRECTIONS.md`); must be E2E-encrypted if ever built.

## Consequences

### Positive

- Works offline; privacy-aligned; export is the backup story.
- Schema versioning enables migrations without vendor lock-in.
- CI can enforce content quality independent of user data.

### Negative / trade-offs

- Multi-device requires manual export/import until sync ships.
- Browser storage quotas require UX for large photo libraries.
- OPFS support varies slightly by browser — fallback required.

## Alternatives considered

| Alternative | Why not |
|-------------|---------|
| Cloud-only DB (Firebase, etc.) | Violates local-first / forced account pressure |
| `localStorage` for records | Quota + no structured indexing + poor binary support |
| File System Access API only | Weaker mobile story; still need in-browser DB for app state |
| SQLite WASM | Strong option later; higher Phase 1 complexity — revisit if IDB pain appears |

## Follow-ups

- Phase 1: implement repositories + ZIP round-trip tests.
- Document storage quota warnings in UI.
